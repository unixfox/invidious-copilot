# GitHub Copilot Instructions for Invidious

This repository contains Invidious, an open-source alternative front-end to YouTube written in Crystal.

## Development Environment Setup

### Crystal Installation (Required Version: 1.16)

**IMPORTANT**: You MUST install Crystal version 1.16 specifically. Other versions will cause compilation failures.

```bash
# Install Crystal using official installer
curl -fsSL https://crystal-lang.org/install.sh | sudo bash

# Install Crystal 1.16 specifically (NOT crystal or crystal 1.16.0)
sudo apt install crystal1.16
```

### Dependencies

Install required system dependencies:

```bash
sudo apt install -y libsqlite3-dev
```

### Project Setup

```bash
# Install Crystal dependencies
make get-libs

# Verify compilation without building binaries
make verify

# Build the project
make invidious

# Run tests
make test

# Format code
make format
```

## YouTube Debugging Strategies

When encountering YouTube-related issues (empty fields, incorrect data, missing hash keys, 0 views, BrokenTubeException, etc.), follow these debugging approaches:

### 1. Create Crystal Debug Scripts

Create small, focused Crystal files that isolate and test specific YouTube API interactions. Structure them like:

```crystal
# debug_youtube_issue.cr
require "http/client"
require "json"

# Extract minimal portions of existing invidious code
# Test specific YouTube endpoint responses
# Add detailed logging and error handling

# Example structure:
client = HTTP::Client.new("www.youtube.com", tls: true)
response = client.get("/some/endpoint?params=values")

puts "Status: #{response.status_code}"
puts "Headers: #{response.headers}"
puts "Body: #{response.body}"

# Parse and validate response structure
begin
  data = JSON.parse(response.body)
  # Test specific fields that are causing issues
rescue ex
  puts "JSON parsing error: #{ex.message}"
end
```

### 2. Available External Tools

#### MCP Fetch Tool
You have access to the MCP fetch tool for sending direct requests to YouTube:
- Use this for testing API endpoints without Crystal overhead
- Compare responses with what Invidious receives
- Validate API changes or inconsistencies

#### YouTube Utils Bash Tool
There's a specialized bash tool available for YouTube API requests:
- Repository: https://github.com/iv-org/youtube-utils/blob/master/scripts/yt-api-helper.sh
- Use for standardized YouTube API testing
- Helpful for reproducing issues in isolation

### 3. Debugging Workflow

1. **Isolate the Issue**: Create a minimal Crystal script that reproduces the problem
2. **Compare Responses**: Use MCP fetch or bash tools to compare expected vs actual responses
3. **Analyze Differences**: Look for API changes, new fields, modified structures
4. **Test Fixes**: Implement changes in isolated scripts before modifying main codebase
5. **Validate**: Ensure fixes don't break existing functionality

## Code Structure and Patterns

### Crystal Language Specifics

- **Type Safety**: Crystal is statically typed - leverage this for catching YouTube API structure changes
- **Macros**: Use Crystal macros for repetitive YouTube data parsing patterns
- **Concurrent HTTP**: Utilize Crystal's fiber-based concurrency for multiple YouTube requests
- **Error Handling**: Use Crystal's exception system effectively for YouTube API errors

### Key Directories

- `src/`: Main application source code
- `spec/`: Test files
- `config/`: Configuration files
- `scripts/`: Utility scripts
- `mocks/`: Mock data for testing

### Build System

- Use `Makefile` for all build operations
- `shard.yml` defines Crystal dependencies
- `.ameba.yml` configures the Crystal linter

## Common YouTube Issue Types

1. **Missing Hash Keys**: Often indicates API structure changes
2. **Zero Views/Incorrect Numbers**: May signal API endpoint modifications
3. **Empty Fields**: Could indicate new required parameters or authentication changes
4. **BrokenTubeException**: Usually network or API availability issues
5. **Parsing Errors**: Often due to new or modified JSON structures

## Development Best Practices

- Always test with Crystal 1.16 to match production environment
- Create focused debug scripts for each YouTube issue
- Use external tools to validate API behavior independently
- Maintain compatibility with existing Invidious API
- Document any YouTube API changes discovered during debugging
- Consider edge cases and error conditions in YouTube responses

## Troubleshooting

If compilation fails:
1. Verify Crystal version: `crystal --version` (should be 1.16.x)
2. Check dependencies: `make get-libs`
3. Try clean build: `make clean && make invidious`
4. Verify system dependencies are installed

For YouTube debugging issues:
1. Start with external tool validation (MCP fetch or bash tool)
2. Create minimal reproduction case
3. Compare with working similar functionality in codebase
4. Test incrementally with small changes
